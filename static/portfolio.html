Project 1

For this project, I implemented a multi-threaded producer-consumer system in C using pthreads and semaphores. I created a producer thread that reads integers from input and inserts them into a circular buffer, while multiple consumer threads retrieve and process these integers, summing their values. To ensure thread safety, I synchronized access to the shared buffer using semaphores and managed race conditions effectively. I also made sure to clean up all dynamically allocated memory, semaphores, and mutexes to ensure efficient resource management. Additionally, I implemented a barrier(n) routine to synchronize all consumer threads at a common point before printing their results and terminating, which fulfilled the extra credit requirement. To measure program performance, I used gettimeofday() to calculate and print the total execution time. I also created a Makefile to streamline the compilation process and manage cleanup efficiently. This project allowed me to demonstrate my understanding of thread synchronization, shared resource management, and program timing.

Project 2

I successfully completed two interrelated projects aimed at solving complex computational problems:

  • Palindrome Recognition: I designed and implemented a recursive Java program called palindromecheck, capable of identifying palindromes in user-input strings. The program supports case-insensitivity and ignores spaces and punctuation, ensuring recognition of iconic examples such as "A man, a plan, a canal: Panama!". This project highlights my proficiency in recursive algorithms and user-focused software design.
  •    Counting Inversions: I implemented two solutions for counting inversions in arrays:
    • easyinversioncount: A brute-force O(n^2) algorithm to establish a baseline.
    • fastinversioncount: An optimized O(n log n) algorithm utilizing enhanced merge sort for efficiency.

These projects emphasized my understanding of recursion, algorithmic optimization, and interactive user input handling. They also underline my commitment to delivering user-friendly and efficient solutions.

Project 3

I created a prototype for an optimized lifting app based on user needs, UX design laws, A/B testing, interviews, and interface principles. The app is called OPTI!

The OPTI app is a user-focused fitness application designed to deliver a minimalist, intuitive, and engaging experience while simultaneously optimizing a users strength, aesthetics, or cardio goals. Through the application of UX Design Laws such as Fitts' Law and Jacob’s Law, changes were made to improve usability, including enlarging buttons for accessibility and relocating key elements for quicker interaction. A/B testing revealed that users preferred a design with larger, better-placed buttons and integrated progress tracking, which enhanced motivation and reduced time-to-action. Interviews with users highlighted the need for personalized, real-time feedback and adaptive workout recommendations, which were incorporated into the app’s design. This iterative process taught the importance of aligning functionality with user expectations and accessibility needs. Ultimately, OPTI empowers users by providing pre- and post-workout feedback, real-time tracking, and live challenges to optimize fitness performance while fostering engagement and community.
